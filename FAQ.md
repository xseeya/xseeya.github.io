# Часто задаваемые вопросы (FAQ)

## Общие вопросы

### ❓ Как установить этот проект?

Просто скачайте или клонируйте репозиторий. Никаких дополнительных зависимостей не требуется - это чистый HTML/CSS/JavaScript!

```bash
git clone https://github.com/username/repository-name.git
cd repository-name
```

Откройте `index.html` в браузере для локального просмотра.

### ❓ Нужен ли мне Node.js или npm?

Нет! Проект написан на чистом JavaScript и не требует никаких инструментов сборки.

### ❓ Как добавить свои музыкальные файлы?

1. Поместите MP3 файлы в папку `music/`
2. Создайте или скачайте LRC файлы для текстов песен и поместите их в `lyrics/`
3. Обновите `playlist.json` с информацией о ваших треках

### ❓ Где взять LRC файлы?

Вы можете:
- Создать их вручную в текстовом редакторе
- Использовать онлайн-генераторы: https://www.lrcgenerator.com/
- Найти готовые на https://lrclib.net/ или https://www.megalobiz.com/lrc/

## Проблемы с развертыванием

### ❓ Сайт не открывается на GitHub Pages

**Решения:**
1. Убедитесь, что в Settings → Pages выбрана правильная ветка (main или gh-pages)
2. Проверьте, что `index.html` находится в корне выбранной папки
3. Подождите 5-10 минут после первого деплоя
4. Очистите кэш браузера (Ctrl+Shift+R)

### ❓ CSS/JS файлы не загружаются (404 ошибка)

**Причина:** Неправильные пути к файлам

**Решение:**
Убедитесь, что пути относительные:
```html
✅ Правильно: <link rel="stylesheet" href="css/styles.css">
❌ Неправильно: <link rel="stylesheet" href="/css/styles.css">
```

### ❓ Музыка не воспроизводится

**Возможные причины:**
1. Файлы не добавлены в репозиторий
2. Неправильные пути в `playlist.json`
3. Файлы слишком большие (GitHub имеет лимит 100MB на файл)
4. Неподдерживаемый формат аудио

**Решения:**
- Проверьте пути в `playlist.json`
- Сожмите аудиофайлы до битрейта 128kbps
- Используйте MP3 или OGG форматы
- Для больших файлов используйте Git LFS

## Функциональность плеера

### ❓ Как работает синхронизация текстов?

Плеер парсит LRC файлы и синхронизирует текст с временем воспроизведения. Формат LRC:
```lrc
[00:12.50]Строка текста
```
Где `00` - минуты, `12` - секунды, `50` - сотые доли секунды.

### ❓ Можно ли кликнуть на строку текста?

Да! Клик по любой строке текста перемотает трек к этому моменту.

### ❓ Как работает визуализация?

Визуализация использует Web Audio API для анализа частот аудио в реальном времени и отображает их на canvas.

### ❓ Поддерживаются ли горячие клавиши?

Да! Нажмите пробел для play/pause (когда фокус не на поле ввода).

## Кастомизация

### ❓ Как изменить цвета?

Откройте `css/styles.css` и измените CSS переменные в секциях `:root` и `[data-theme="dark"]`.

Подробнее в файле `CUSTOMIZATION.md`.

### ❓ Как добавить больше социальных сетей?

Добавьте новые ссылки в секцию `.social-links` в `index.html`. Иконки можно взять с:
- https://heroicons.com/
- https://fontawesome.com/
- https://feathericons.com/

### ❓ Можно ли изменить аватар?

Да! Замените файл `assets/avatar.svg` на свое изображение (JPG, PNG или SVG).

### ❓ Как добавить фоновое изображение?

Добавьте в `css/styles.css`:
```css
body {
    background-image: url('../assets/background.jpg');
    background-size: cover;
    background-position: center;
    background-attachment: fixed;
}
```

## Технические вопросы

### ❓ Какие браузеры поддерживаются?

- Chrome/Edge 90+
- Firefox 88+
- Safari 14+
- Opera 76+

Старые браузеры могут иметь проблемы с ES6 модулями или Web Audio API.

### ❓ Почему используется ES6 модули?

ES6 модули позволяют лучше организовать код без необходимости в сборщиках (webpack, rollup).

### ❓ Можно ли добавить React/Vue?

Технически да, но это потребует настройки сборки. Текущая версия работает без зависимостей для максимальной простоты.

### ❓ Как протестировать локально?

**Вариант 1: Простой HTTP сервер**
```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000

# Node.js
npx http-server
```

Затем откройте http://localhost:8000

**Вариант 2: VS Code Live Server**
Установите расширение "Live Server" и нажмите "Go Live"

### ❓ Почему не работает при открытии file:///?

ES6 модули требуют HTTP протокол из-за политики CORS. Используйте локальный веб-сервер для тестирования.

## Оптимизация

### ❓ Как уменьшить размер аудиофайлов?

```bash
# Используйте FFmpeg для сжатия:
ffmpeg -i input.mp3 -b:a 128k output.mp3
```

### ❓ Как ускорить загрузку сайта?

1. Сожмите изображения (https://tinypng.com/)
2. Уменьшите битрейт аудио до 128kbps
3. Используйте WebP для изображений
4. Минифицируйте CSS/JS (опционально)

### ❓ Что делать с большими файлами?

Для файлов > 50MB используйте:
1. Git LFS (Large File Storage)
2. Внешний хостинг для аудио (SoundCloud API, YouTube)
3. CDN сервисы
